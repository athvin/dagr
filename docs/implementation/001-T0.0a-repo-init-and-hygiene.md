# 001 · T0.0a — Repository initialization and hygiene

> **Milestone:** M0 · **Size:** S · **Type:** setup · **Components:** system-level (Stability, supply chain)
> **Branch:** `chore/t0.0a-repo-init-and-hygiene` · **Depends on:** — · **Blocks:** T1, T7

## Why / context
dagr's entire pitch is compile-time confidence and a stable, auditable artifact trail (arch.md "What this is", "Stability"). Before a single crate exists, the repo needs the invariants an L7 Rust engineer sets up first: a pinned toolchain so builds and UI-test snapshots are reproducible across machines, an explicit lint policy so "clippy with warnings denied" is a shared contract rather than a per-developer habit, and the licensing/formatting/ignore hygiene that every later ticket assumes. This ticket is pure scaffolding: it establishes those repo-level files so that T1 (the Cargo workspace skeleton) drops crates into a governed tree and T7 (the CI pipeline) has a toolchain, a fmt config, and a deny set to enforce. No crates and no `Cargo.toml` are created here — the workspace itself is T1.

## Objective
Stand up the greenfield repository hygiene layer that governs every crate added later, without adding any crate.

Concrete pieces of work:
- A pinned toolchain manifest (`rust-toolchain.toml`) fixing the channel/version and the components CI relies on (`rustfmt`, `clippy`), so the workspace MSRV commitment (arch.md "Stability") has a single source of truth and UI snapshots are toolchain-deterministic (supports T8's pinned-toolchain requirement).
- A `rustfmt.toml` capturing the project's formatting policy so `cargo fmt --check` is meaningful and stable.
- A workspace-wide **lint policy** document/config: the clippy deny set (warnings-denied posture) plus any explicitly-allowed exceptions, in a form T1/T7 can apply workspace-wide (for example lint groups the workspace manifest will inherit, plus a short rationale note for each allowed exception). This ticket authors the *policy*; T1 wires it into `Cargo.toml` and T7 enforces it in CI.
- A `LICENSE` file (project-chosen license) at the repo root, and license metadata expectations recorded so `cargo deny`'s license checks in T7 have a target to validate against.
- A top-level `README` stub: name, one-line description, the permanent "what it is not" boundary, a placeholder for the quickstart (arch.md "Documentation" system criterion 1) and an explicit MSRV line placeholder that arch.md "Stability" requires the README to carry.
- An `.editorconfig` encoding whitespace/line-ending/charset defaults consistent with `rustfmt.toml`.
- Reconcile the **existing** `.gitignore`: confirm it covers Cargo build output and rustfmt backups (it does today), and extend it to ignore local run-store / scratch / artifact output directories that later milestones write, plus common editor/OS noise, so generated artifacts never get committed.

## Test plan (write these first — TDD)
These are hygiene invariants, not unit tests; each is a concrete, independently-checkable condition. Author them as the acceptance gate before creating the files, then make each pass.

1. **Toolchain is pinned and self-consistent.** Setup: a clean checkout of the branch on a machine with `rustup` and no other override. Action: read `rust-toolchain.toml` and run the toolchain resolver (for example `rustup show`). Expected: a specific, non-`stable`-floating version/channel is resolved, `rustfmt` and `clippy` components are listed, and the resolved version matches the MSRV line written in the README (no drift between the two).
2. **Formatting policy applies cleanly to a trivial input.** Setup: a throwaway Rust source snippet placed in a scratch location (not committed). Action: run `cargo fmt` (or `rustfmt`) against it using the repo's `rustfmt.toml`. Expected: the config is accepted with no unknown-option errors, and a deliberately mis-formatted snippet is reported by `--check` while an already-formatted one passes — proving the config is live, not inert.
3. **Lint policy names a deny set and justifies every exception.** Setup: open the lint-policy artifact. Action: read it. Expected: it states the warnings-denied posture explicitly, enumerates the denied lint groups/levels, and every `allow` exception carries a one-line rationale; there is a documented, single place T1 will reference to apply it workspace-wide. Cross-check: no exception silently weakens the "clippy with warnings denied" contract that the Definition of done's final item enforces.
4. **License is present and machine-readable.** Setup: repo root. Action: locate `LICENSE` and read the recorded license identifier. Expected: a standard SPDX license identifier is present and matches what the license-metadata note tells T7's `cargo deny` to allow, so the supply-chain license check has an unambiguous target.
5. **README states the boundary and the MSRV.** Setup: repo root. Action: read `README`. Expected: it contains the project name, a one-line description, the permanent non-goals ("not a scheduler, distributed execution system, metadata store, web interface, DSL, or backfill orchestrator; graph shape never changes at runtime"), an explicit MSRV line, and a clearly-marked placeholder for the quickstart — with nothing that claims a feature the code does not yet have.
6. **EditorConfig and rustfmt agree.** Setup: open both files. Action: compare indent style/size, end-of-line, final-newline, and charset. Expected: no contradiction between `.editorconfig` and `rustfmt.toml`; an editor honoring `.editorconfig` produces output `rustfmt --check` accepts.
7. **Gitignore hides all generated output.** Setup: from a clean tree, create representative generated paths (a `target/` entry, a `*.rs.bk`, and a placeholder run-store/scratch/artifact directory of the shape later milestones write). Action: run `git status --porcelain` and `git check-ignore` on each path. Expected: every generated path is ignored; no such path appears as untracked. Negative check: a normal source path is *not* ignored.
8. **No crate leaked in.** Setup: repo root after this ticket. Action: search the tree for `Cargo.toml`, `Cargo.lock`, `src/`, or any `.rs` file. Expected: none exist — this ticket adds hygiene only; the workspace is T1.

## Definition of done
- [ ] `rust-toolchain.toml` pins a specific toolchain version/channel and declares the `rustfmt` and `clippy` components (arch.md "Stability": MSRV pinned in the workspace).
- [ ] The pinned toolchain version is reflected by an explicit **MSRV** line in the `README`, with no drift between the two (arch.md "Stability": MSRV documented in the README).
- [ ] `rustfmt.toml` exists, is accepted by `rustfmt` with no unknown-option errors, and a mis-formatted snippet fails `--check` while a formatted one passes.
- [ ] A workspace-wide **lint policy** exists that states the warnings-denied posture, enumerates the clippy deny set, and gives a one-line rationale for every allowed exception, in a form T1 can inherit workspace-wide and T7 can enforce (supports "clippy with warnings denied").
- [ ] `LICENSE` exists at the repo root with a standard SPDX identifier, and a license-metadata note records the identifier T7's `cargo deny` license check will allow (arch.md "Stability": supply chain — `cargo deny` licenses).
- [ ] `README` stub carries: project name, one-line description, the permanent non-goals boundary (arch.md "What this is": not a scheduler/distributed system/metadata store/web interface/DSL/backfill orchestrator; graph shape never changes at runtime), the MSRV line, and a marked placeholder for the quickstart (arch.md "Documentation" / system criterion 1).
- [ ] `README` names the supported platform tiers accurately or defers them to a placeholder consistent with arch.md "Platform support" (Tier 1 Linux containers; dev-supported macOS; Windows unsupported in v1) — no claim of Windows support.
- [ ] `.editorconfig` exists and does not contradict `rustfmt.toml` on indent, EOL, final newline, or charset.
- [ ] The existing `.gitignore` is reconciled and extended to cover Cargo build output, rustfmt backups, and the local run-store/scratch/artifact output shapes later milestones write, plus common editor/OS noise; verified via `git check-ignore` on representative paths.
- [ ] No `Cargo.toml`, `Cargo.lock`, `src/`, or `.rs` file is introduced by this ticket (workspace is T1).
- [ ] The lint-policy and license-metadata artifacts are placed where T7's CI (`cargo audit` / `cargo deny`, fmt, clippy) and T1's workspace manifest will reference them, so no downstream ticket has to invent the location.
- [ ] CI is green on the ticket branch (fmt, clippy with warnings denied, tests, rustdoc lint, and cargo-audit/deny where configured).

## Out of scope
- The Cargo workspace, any crate (`core`, `artifact`, `render`, `cli`), placeholder lib targets, `Cargo.toml`, or `Cargo.lock` — that is **T1**.
- The GitHub Actions pipeline, the acceptance-criteria coverage matrix, and actually running fmt/clippy/audit/deny in CI — that is **T7**; this ticket only authors the toolchain, fmt config, lint policy, and license target those jobs consume.
- The trybuild/UI compile-failure harness and snapshot flow — that is **T8**; this ticket only pins the toolchain that makes its snapshots deterministic.
- The full stability policy (semver rules, schema-evolution, fixture-corpus plan, fingerprint-algorithm versioning, criteria partition) — that is **T0.10**; this ticket carries only the MSRV pin and the supply-chain license/deny target that T0.10 and T7 build on.
- The README quickstart's actual two-node walkthrough and any runnable examples or cookbook — later documentation tickets; here they are placeholders only.
- Any scope-boundary temptation: do not scaffold config for a server, scheduler, metadata store, web UI, DSL, or backfill/runtime-graph-mutation feature — none of those exist in dagr and no hygiene file should imply them.
